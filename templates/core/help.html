{% trans "¿Qué es Jira (y qué es jirrabit)?" %}

{% blocktrans %} Jira es una herramienta para coordinar el trabajo de un equipo: anotar tareas, asignarlas, priorizarlas, seguir su avance y saber qué falta para terminar. jirrabit es un clon ligero con los conceptos esenciales: proyectos, sprints, epics, issues, comentarios, adjuntos y un tablero kanban. {% endblocktrans %}

{% trans "Glosario" %}

{% trans "Proyecto" %}

{% blocktrans %} Contenedor de todo el trabajo de un producto o equipo. Cada proyecto tiene una clave corta (por ejemplo WEB) que sirve de prefijo para todas sus tareas (WEB-1, WEB-2…). Tiene un lead y una lista de miembros con su rol (admin, member o viewer). {% endblocktrans %}

{% trans "Issue (tarea)" %}

{% blocktrans %} Unidad básica de trabajo: una cosa que alguien tiene que hacer. Cada issue tiene una clave única, un resumen, una descripción (Markdown soportado), un estado, una prioridad, un tipo, quien la reporta y quien la tiene asignada. {% endblocktrans %}

  • {% trans "Story" %} — {% trans "Funcionalidad nueva contada desde el punto de vista del usuario." %}
  • {% trans "Task" %} — {% trans "Tarea técnica o trabajo concreto sin valor directo de negocio." %}
  • {% trans "Bug" %} — {% trans "Algo que no funciona como debería." %}
  • {% trans "Subtask" %} — {% trans "Pieza más pequeña que cuelga de otro issue (su parent)." %}

{% trans "Epic" %}

{% blocktrans %} Pieza grande de trabajo que agrupa varios issues relacionados. Una epic puede ser por ejemplo «Checkout v2» y contener múltiples stories y tasks que en conjunto la completan. {% endblocktrans %}

{% trans "Sprint" %}

{% blocktrans %} Ventana de tiempo (típicamente 1 a 4 semanas) en la que el equipo se compromete a terminar cierto conjunto de issues. Tiene estado future (planificado), active (en curso) o closed (terminado). {% endblocktrans %}

{% trans "Estado y workflow" %}

{% blocktrans %} El estado describe en qué punto está un issue: por ejemplo To Do, In Progress, In Review, Blocked, Done. Cada estado pertenece a una categoría (todo, in_progress, done) que el tablero usa para agrupar las columnas. Los administradores pueden restringir las transiciones permitidas (por ejemplo, exigir pasar por In Review antes de Done). {% endblocktrans %}

{% trans "Prioridad" %}

{% blocktrans %} Indica la urgencia: Highest, High, Medium, Low, Lowest. Se usa para ordenar la cola de trabajo y filtrar listados. {% endblocktrans %}

{% trans "Story points" %}

{% blocktrans %} Estimación abstracta del esfuerzo de un issue (no de horas). Sirven para calcular velocidad por sprint y dibujar el burndown chart. {% endblocktrans %}

{% trans "Etiquetas (labels)" %}

{% blocktrans %} Tags libres que se pueden añadir a un issue para clasificarlo (frontend, backend, qa…). Un issue puede tener varias. {% endblocktrans %}

{% trans "Asignado, reporter y watchers" %}

  • {% trans "Reporter" %} — {% trans "quien creó el issue." %}
  • {% trans "Asignado (assignee)" %} — {% trans "persona responsable de avanzarlo." %}
  • {% trans "Watchers" %} — {% trans "gente suscrita a los cambios; reciben emails y notificaciones." %}

{% trans "Comentarios y menciones" %}

{% blocktrans %} Cualquier miembro del proyecto puede comentar en un issue. Soporta Markdown y permite mencionar a otros usuarios con @usuario; la persona mencionada recibe una notificación. {% endblocktrans %}

{% trans "Adjuntos" %}

{% blocktrans %} Archivos pequeños asociados al issue (capturas, logs, notas…). Se guardan en base64 dentro de la base de datos; tamaño máximo 5 MB. {% endblocktrans %}

{% trans "Time tracking" %}

{% blocktrans %} Cada issue tiene una estimación (en minutos) y se le pueden ir registrando work logs según se avanza. El panel lateral muestra tiempo registrado / estimado y la lista de logs recientes. {% endblocktrans %}

{% trans "Issue links" %}

{% blocktrans %} Relaciones entre issues: bloquea, bloqueado por, relacionado con, duplica, duplicado por. Al crear un link, jirrabit añade automáticamente el inverso en el otro issue. {% endblocktrans %}

{% trans "Flujos típicos" %}

{% trans "Crear un proyecto" %}

  1. {% trans "Pulsa «Nuevo proyecto» en la lista de proyectos." %}
  2. {% trans "Asigna clave corta (3-5 letras) y nombre largo." %}
  3. {% trans "Añade miembros desde la pestaña «Miembros» con su rol." %}

{% trans "Crear y trabajar una tarea" %}

  1. {% trans "Pulsa «+ Crear tarea» dentro de un proyecto." %}
  2. {% trans "Rellena resumen, tipo, prioridad, sprint (opcional) y descripción." %}
  3. {% trans "Una vez creada, los campos se editan in-line haciendo click sobre ellos." %}
  4. {% trans "Mueve el estado desde el tablero arrastrando la tarjeta o desde el detalle." %}

{% trans "Planificar un sprint" %}

  1. {% trans "Crea el sprint desde el detalle del proyecto." %}
  2. {% trans "Asigna issues al sprint desde el backlog o desde el detalle del issue." %}
  3. {% trans "Cuando comience, pulsa «Iniciar» en el sprint." %}
  4. {% trans "Sigue el avance en el tablero y revisa la burndown chart en cualquier momento." %}

{% trans "Buscar con JQL-lite" %}

{% blocktrans %} En la barra de búsqueda usa una mini-sintaxis tipo Jira. Ejemplos: {% endblocktrans %}

  • project=WEB AND status="In Progress"
  • assignee=alice AND priority=High ORDER BY updated DESC
  • label=frontend AND type=Bug
  • epic="Checkout v2" AND status!=Done

{% blocktrans %} Campos soportados: project, status, priority, assignee, reporter, type, label, sprint, epic, text. Operadores: =, !=, ~ (contiene), in (a,b). Para los campos de usuario también vale el display name (assignee="Erin Soto"). {% endblocktrans %}
